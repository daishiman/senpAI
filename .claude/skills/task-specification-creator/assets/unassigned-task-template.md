# 未完了タスク指示書テンプレート

> **Progressive Disclosure**
> - 読み込みタイミング: Phase 12（ドキュメント更新）での未タスク検出時
> - 読み込み条件: MINOR判定された指摘事項を未完了タスク指示書に変換するとき
> - 使用スキーマ: schemas/unassigned-task.json
> - 関連エージェント: agents/generate-unassigned-task.md
> - 関連スクリプト: scripts/detect-unassigned-tasks.js

このテンプレートは「指示書」として機能する。
受け手（人間またはAI）がこのドキュメントを読むだけで、
タスクの全体像を理解し、具体的なアクションを実行できる粒度で記述する。
**100人中100人が同じ理解でタスクを実行できること**を目標とする。

---

## 配置先

```
docs/30-workflows/unassigned-task/{{ファイル名}}.md
```

## 品質チェックリスト

未完了タスク指示書を作成する際、以下の品質基準を満たすこと:

### Why（なぜ必要か）【必須】

- [ ] 背景が明確に記述されている
- [ ] 問題点・課題が具体的に説明されている
- [ ] 放置した場合の影響が記載されている

### What（何を達成するか）【必須】

- [ ] 目的が具体的に定義されている
- [ ] 最終ゴールが検証可能な形で記述されている
- [ ] スコープが明確に定義されている（含むもの/含まないもの）
- [ ] 成果物が列挙されている

### How（どのように実行するか）【必須】

- [ ] 前提条件・依存関係が明記されている
- [ ] 手順が100人中100人が同じ理解で実行できる粒度で記述されている
- [ ] 完了条件が検証可能な形で定義されている
- [ ] テストケース/検証方法が記載されている

### Phase構成【必須】

- [ ] タスクの規模に応じたPhase構成が定義されている
- [ ] 各Phaseに目的・手順・成果物・完了条件が記載されている
- [ ] Phase間の依存関係が明確になっている

### リスクと対策【必須】

- [ ] 想定されるリスクが洗い出されている
- [ ] 各リスクの影響度・発生確率が評価されている
- [ ] 各リスクに対する対策が記載されている
- [ ] 過去の類似タスクで発生した問題（06-known-pitfalls.md）を確認済み

### 参照情報【推奨】

- [ ] 関連ドキュメント（aiworkflow-requirements/references/）へのリンクが記載されている
- [ ] 関連する既知の落とし穴（Pitfall ID）が明記されている
- [ ] 参考となる完了済みタスク（completed-tasks/）が参照されている
