## 苦戦箇所と解決策

### 苦戦1: 複数タスクの並行マージコンフリクト検出

**問題**:
Phase-12 の成果物ファイル（`outputs/phase-12/skill-feedback-report.md` 等）に
`||||||| Stash base` マーカーが出現しており、複数タスクからの同時マージによる
コンフリクト状態が検出された。

**再発条件**:
- 複数のワークツリーが同一ファイルを同時に編集している場合
- Phase-12 / Phase-11 成果物ファイルは複数タスクで共有されやすい

**検出方法**:
```bash
# コンフリクトマーカーを全成果物ファイルで検索
grep -r "<<<<<<\|>>>>>>>\||||||| Stash base" outputs/
```

**解決策**:
1. Phase-13 PR 作成前に必ずコンフリクトマーカーの有無を確認する
2. `outputs/` 配下に `||||||| Stash base` が存在する場合はマージを中止し、手動解消する
3. 成果物ファイルを per-task ディレクトリ（`outputs/phase-12/<task-id>/`）に
   分離することで衝突確率を下げる

**将来の参照方法**:
Phase-13（PR作成）前のチェックリストに「コンフリクトマーカー検索」を追加する。

### 苦戦2: task-workflow.md インデックステーブル全行への「区分」列一括追加の手間

**問題**:
Phase 5 での実装時に、25行以上のテーブルに対して行ごとに区分値
（正本 / 履歴 / 契約仕様）を判断しながら追記する手順が必要だった。
各行の意味を読んで区分を判断する作業は、テーブルが長くなるほど工数が増大する。

**再発条件**:
- インデックステーブルが20行以上の場合
- 新規列を全行に追加する場合

**解決策**:
1. **短いテーブル（10行以下）**: 手動で行ごとに追記する
2. **長いテーブル（11行以上）**: 置換スクリプトを事前に用意してから実施する

**スクリプト例（区分列を追加する場合）**:
```bash
# テーブルヘッダーに列追加
sed -i 's/| ファイル | 役割 |/| ファイル | 役割 | 区分 |/' target.md

# 行ごとに区分を設定する場合は jq や awk でマッピングテーブルを用意する
```

3. テーブル設計時点で「区分」列を含める（後付けを避ける）

**将来の参照方法**:
新しいインデックステーブルを設計する際は、最初から「区分」列を含むテンプレートを使う。
`lessons-learned-templates.md` のテーブルテンプレートに「区分」列を追加しておく。

### 苦戦3: 正本・履歴判別の属人化解消

**問題**:
UT-VERIFY-DOC-CONSOLIDATION-001 実施前は、verify 関連ドキュメント群（4ファイル）の
正本・履歴の区別がファイルを開かなければわからなかった。
これは新規参加者や長期未参照後の作業者にとって混乱の原因となっていた。

**症状**:
- `task-workflow.md`（インデックス）と `task-workflow-active.md`（正本）が混在
- `task-workflow-completed.md`（履歴）と `interfaces-skill-verify-contract.md`（契約仕様）が
  同一ディレクトリに並列配置されており、役割が不明確

**解決策**:
各ファイルの冒頭に統一形式のラベルを付与する：

```markdown
> 区分: 正本（current contract）
```

```markdown
> 区分: 履歴記録（history record）
```

```markdown
> 区分: 契約仕様
```

**統一形式の利点**:
- ファイルを開いた瞬間に区分が判別できる
- IDE の preview や GitHub の web UI でもすぐに視認できる
- `grep "区分:" *.md` で全ファイルの区分を一覧できる

**将来の参照方法**:
新規ドキュメントを作成するときは冒頭に `> 区分: XXX` を付与するルールとする。
区分値の候補: `正本（current contract）` / `履歴記録（history record）` / `契約仕様` /
`インデックス` / `テンプレート`

## 知見まとめ

| 1   | **コンフリクトマーカー検索を PR 前チェックに追加** | 複数ワークツリーが同一ファイルを編集する並行作業時          |
