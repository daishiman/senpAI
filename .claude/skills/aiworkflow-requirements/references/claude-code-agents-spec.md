## YAML Frontmatter 必須フィールド

| フィールド    | 型     | 必須 | 制約                                   | 説明                                       |
| ------------- | ------ | ---- | -------------------------------------- | ------------------------------------------ |
| `name`        | string | 必須 | 小文字・数字・ハイフンのみ、最大64文字 | エージェント識別子（ケバブケース）         |
| `description` | string | 必須 | 最大1024文字                           | エージェントの説明（依存スキル一覧を含む） |

## YAML Frontmatter オプションフィールド

| フィールド | 型       | 説明                                     |
| ---------- | -------- | ---------------------------------------- |
| `tools`    | string[] | 使用可能ツールのリスト（省略時は全継承） |
| `model`    | string   | 使用モデル（sonnet/opus/haiku/inherit）  |
| `version`  | string   | バージョン番号（メタデータ、非公式）     |

---

## 完全な YAML Frontmatter 記述形式

YAML Frontmatterは以下の構造で記述する。

| 行位置 | 記述内容                                                     |
| ------ | ------------------------------------------------------------ |
| 1行目  | 区切り線 `---`                                               |
| 2行目  | `name:` フィールド（ケバブケースのエージェント名）           |
| 3行目  | `description:` フィールド開始（`|` でマルチライン指定）      |
| 4-6行目| エージェントの目的と専門分野（2-3行でインデント）            |
| 7行目  | 空行                                                         |
| 8行目  | モデル人物の記述（実在する専門家名と専門分野）               |
| 9行目  | 空行                                                         |
| 10行目 | 発動条件（`Use proactively when` で始まる英語文）            |
| 11行目 | `tools:` フィールド開始                                      |
| 12-N行目| ツールリスト（各行 `- tool-name` 形式でインデント）         |
| 最終-1行| `model:` フィールド（sonnet/opus/haiku/inherit）           |
| 最終行 | 区切り線 `---`                                               |

**フィールド記述ルール**:

| フィールド    | 記述ルール                                              |
| ------------- | ------------------------------------------------------- |
| `name`        | ケバブケース、エージェントの役割を表す名前              |
| `description` | 簡潔な目的説明 + モデル人物 + 依存スキル一覧 + 発動条件 |
| `tools`       | 必要なツールのみ列挙（省略時は全ツール継承）            |
| `model`       | タスクの複雑さに応じて選択（デフォルト: sonnet）        |

---

## description フィールドの詳細記述規則

**目的**: description フィールドに依存スキルを明確に記述し、エージェントがスキルを確実に参照できるようにする。

**記述形式**:

description フィールドは以下の4ブロックで構成される。

| ブロック     | 行数   | 記述内容                                                   |
| ------------ | ------ | ---------------------------------------------------------- |
| 目的・専門分野 | 2-3行  | エージェントの目的と専門分野を簡潔に記述                   |
| 空行         | 1行    | ブロック間の区切り                                         |
| モデル人物   | 1行    | 「モデル人物: 実在する専門家名 - 専門分野」の形式で記述    |
| 空行         | 1行    | ブロック間の区切り                                         |
| 発動条件     | 1行    | 「Use proactively when」で始まる英語での発動条件           |

**記述の原則**:

| 原則                   | 説明                                                       |
| ---------------------- | ---------------------------------------------------------- |
| 簡潔性                 | 目的説明は2-3行、依存スキル一覧はパスと用途のみ            |
| Progressive Disclosure | スキル内部の詳細はSKILL.mdを読めば分かるため、ここでは不要 |
| 最大500文字            | description の文字数制限を遵守                             |

## 依存スキルの記述規則

**目的**: エージェントがスキルを確実に参照し、タスクを実行できるようにする。

**スキルの役割**:

| 要素           | 説明                                                 |
| -------------- | ---------------------------------------------------- |
| 知識           | ドメイン固有の専門知識、判断基準、ベストプラクティス |
| 実行手順       | Phase別のステップバイステップの実行フロー            |
| 詳細な実行内容 | スクリプト、テンプレート、リソースファイル           |

**記述規則**:

| 規則                 | 説明                                                |
| -------------------- | --------------------------------------------------- |
| 相対パス必須         | `.claude/skills/{{skill-name}}/SKILL.md` 形式で記述 |
| スキル数明記         | `📚 依存スキル ({{N}}個):` で総数を明記             |
| 用途説明必須         | 各スキルに20-40文字の用途説明を付与                 |
| 知識・実行内容を明示 | スキルが提供する「知識」と「実行内容」を記述        |
| Phase別マッピング    | 本文でどのPhaseでどのスキルを読み込むか明記         |

**エージェントとスキルの関係**:

エージェントはタスク実行時に複数のスキルを読み込み、各スキルから必要な情報を取得する。

| 階層                     | 説明                                                         |
| ------------------------ | ------------------------------------------------------------ |
| エージェント（親）       | タスク実行者として、必要なスキルを順次読み込む               |
| スキル読み込み（子）     | 各スキルから以下の3種類の情報を取得する                      |
| - 知識の取得             | 判断基準、ベストプラクティス等のドメイン知識                 |
| - 実行手順               | Phase別のステップバイステップフロー                          |
| - scripts/ の実行        | スクリプトファイルによる確実な処理の実行                     |

エージェントは複数のスキル（スキルA、スキルB等）を同様の方法で読み込み、タスクを完遂する。

## 本文の必須セクション

エージェントファイルの本文は以下のセクションで構成される。

| セクション   | 必須 | 説明                                 |
| ------------ | ---- | ------------------------------------ |
| 目的         | 必須 | エージェントの行動目的               |
| 責務         | 必須 | エージェントの責務                   |
| 背景         | 必須 | エージェントの行動の背景             |
| 役割定義     | 必須 | モデル人物と参考文献                 |
| ワークフロー | 必須 | Phase別の実行フロー                  |

### 役割定義セクションの記述形式

エージェントファイルの本文は以下の見出し構造で記述する。

| 見出しレベル | 見出し名     | 記述内容                                                     |
| ------------ | ------------ | ------------------------------------------------------------ |
| H1 (`#`)     | エージェント名 | エージェントの識別名（ファイル名と一致）                     |
| 本文         | 概要説明     | エージェントの概要を1-2段落で記述                            |
| H2 (`##`)    | 目的         | エージェントの行動目的を明確に記述                           |
| H2 (`##`)    | 責務         | エージェントが担う責務を列挙                                 |
| H2 (`##`)    | 背景         | エージェントの行動の背景・文脈を説明                         |
| H2 (`##`)    | 役割定義     | モデル人物と参考文献を含むセクション                         |
| H3 (`###`)   | モデル人物   | 「実在する専門家名: 専門分野」の形式で記述                   |
| H3 (`###`)   | 参考文献     | 参考とする書籍名と、その書籍の考え方がエージェントの思考・判断にどう活用されるかを記述 |

**参考文献の記述形式**: 各参考文献は「書籍名: 活用方法」の形式で記述し、複数の書籍がある場合は行を分けて列挙する。

## 行数制約

| 項目             | 制約                                                    |
| ---------------- | ------------------------------------------------------- |
| エージェント本文 | 450-550行を目標                                         |
| 超過時の対応     | 詳細知識をスキルに分離する                              |
| 最大行数         | 550行を超えないこと超える場合は、エージェントを分離する |

## 命名規則

### エージェント名

| 規則         | 説明                                       |
| ------------ | ------------------------------------------ |
| 役割ベース   | 役割を表す名前（`{{role}}-{{specialty}}`） |
| ケバブケース | 単語間はハイフンで接続                     |
| 単数形       | 複数形は避ける                             |

## ファイル参照形式

### 相対パスの記述規則

**必須形式**: プロジェクトルートからの相対パス（`.claude/` で始まる形式）

| 参照対象     | パス形式                                 |
| ------------ | ---------------------------------------- |
| エージェント | `.claude/agents/{{agent-name}}.md`       |
| スキル       | `.claude/skills/{{skill-name}}/SKILL.md` |

### 禁止される参照形式

| 形式                        | 問題点                                 |
| --------------------------- | -------------------------------------- |
| スキル名/エージェント名のみ | パスが不明確で読み込みできない         |
| 絶対パス                    | 環境依存（ユーザーごとにパスが異なる） |
| `../` を含む相対パス        | 曖昧さが生じ、正確なパス解決ができない |
