## UI設計

### レイアウト

| パネル       | 配置     | サイズ         |
| ------------ | -------- | -------------- |
| 履歴一覧     | 左サイド | 幅1/3（柔軟）  |
| 詳細パネル   | メイン   | 幅2/3（柔軟）  |
| 復元ダイアログ | オーバーレイ | 中央配置    |

### 表示要素

| 要素             | 表示内容                        |
| ---------------- | ------------------------------- |
| バージョン番号   | v1, v2, v3...（新しい順）       |
| 作成日時         | YYYY/MM/DD HH:mm形式            |
| ファイルサイズ   | 人間可読形式（KB, MB）          |
| 現在バージョン   | 「現在」バッジで識別            |
| ログレベル       | info/warn/error/debugのバッジ   |

### ユーティリティ関数

#### formatDate関数

日時文字列を日本語ロケールでフォーマットする関数。

| 項目       | 内容                                          |
| ---------- | --------------------------------------------- |
| 関数名     | formatDate                                    |
| 入力       | dateString（ISO 8601形式の日時文字列）        |
| 出力       | フォーマット済み文字列（例: 2026/01/26 14:30）|
| ロケール   | ja-JP                                         |
| 出力形式   | YYYY/MM/DD HH:mm                              |

**フォーマットオプション**

| オプション | 値        |
| ---------- | --------- |
| year       | numeric   |
| month      | 2-digit   |
| day        | 2-digit   |
| hour       | 2-digit   |
| minute     | 2-digit   |

#### formatSize関数

バイト数を人間可読形式に変換する関数。

| 項目       | 内容                               |
| ---------- | ---------------------------------- |
| 関数名     | formatSize                         |
| 入力       | bytes（数値、バイト単位）          |
| 出力       | フォーマット済み文字列             |

**変換ルール**

| 条件                | 出力形式             | 例         |
| ------------------- | -------------------- | ---------- |
| 1024バイト未満      | {bytes} B            | 512 B      |
| 1MB未満             | {KB値, 小数点1桁} KB | 45.2 KB    |
| 1MB以上             | {MB値, 小数点1桁} MB | 1.5 MB     |

## アクセシビリティ

### WCAG 2.1 AA準拠

| 要件                 | 実装方針                              |
| -------------------- | ------------------------------------- |
| キーボードナビゲーション | Tab, Enter, Space, Escapeで全操作可能 |
| スクリーンリーダー   | 適切なrole/aria属性を設定             |
| フォーカス管理       | ダイアログ表示時はフォーカストラップ  |
| 色に頼らない情報     | アイコン+テキストで状態を表現         |

### ARIA属性

| コンポーネント | 属性                               |
| -------------- | ---------------------------------- |
| VersionHistory | role="list"                        |
| VersionHistoryItem | role="listitem", aria-label    |
| RestoreDialog  | role="dialog", aria-modal="true"   |
| ConversionLogs | role="list"                        |
| LogEntry       | role="listitem"                    |
| LoadingSkeleton | role="status", aria-label="読み込み中" |
| ErrorDisplay   | role="alert"                       |

### 実装パターン

#### ローディングスケルトン

読み込み中の状態を視覚的に表現するスケルトンUIの実装パターン。

| 項目               | 内容                                   |
| ------------------ | -------------------------------------- |
| コンテナ要素       | div（role="status", aria-label="読み込み中"） |
| スケルトン数       | 3件表示                                |
| スケルトン高さ     | 16px（h-16）                           |
| アニメーション     | animate-pulse（点滅効果）              |
| 背景色             | gray-200                               |
| スクリーンリーダー | sr-onlyクラスで「読み込み中...」を読み上げ |

**属性設定**

| 要素               | 属性                    | 目的                   |
| ------------------ | ----------------------- | ---------------------- |
| コンテナ           | role="status"           | 状態変化を通知         |
| コンテナ           | aria-label="読み込み中" | 状態を説明             |
| スケルトン要素     | aria-hidden="true"      | 読み上げ対象から除外   |
| テキスト要素       | className="sr-only"     | 視覚的に非表示で読み上げ |

#### 履歴アイテムボタン

履歴一覧の各アイテムをボタン要素として実装するパターン。

| 項目           | 内容                                           |
| -------------- | ---------------------------------------------- |
| 要素タイプ     | button（type="button"）                        |
| クリックハンドラ | onSelect関数を呼び出し、選択アイテムを渡す   |
| aria-label     | 「バージョン {version}」形式（最新の場合は「(最新)」を追加） |

**aria-labelの動的生成ルール**

| 条件                 | aria-label値               |
| -------------------- | -------------------------- |
| 通常バージョン       | 「バージョン v1」          |
| 最新バージョン       | 「バージョン v3 (最新)」   |

## エラーハンドリング

### エラー種別

| エラー種別     | 表示メッセージ                     |
| -------------- | ---------------------------------- |
| NetworkError   | 「通信エラーが発生しました」       |
| NotFoundError  | 「データが見つかりません」         |
| RestoreError   | 「復元に失敗しました」             |
| DBError        | 「データベースエラーが発生しました」|
| API未利用可能  | 「History API not available」      |

### エラー表示パターン

| 状態           | 表示方法                        |
| -------------- | ------------------------------- |
| 初期取得失敗   | 全体エラーメッセージ+再試行ボタン |
| 追加読み込み失敗 | インラインエラー+再試行ボタン   |
| 復元失敗       | ダイアログ内エラー表示          |

### ErrorDisplayコンポーネント仕様

エラー状態を表示し、再試行機能を提供するコンポーネント。

#### Props

| Prop名   | 型         | 必須 | 説明                     |
| -------- | ---------- | ---- | ------------------------ |
| message  | string     | Yes  | 表示するエラーメッセージ |
| onRetry  | () => void | Yes  | 再試行ボタン押下時のコールバック |

#### UI構成

| 要素               | 内容                                    |
| ------------------ | --------------------------------------- |
| コンテナ           | div（role="alert"）                     |
| コンテナスタイル   | 角丸、赤系背景（bg-red-50）、パディング4、中央揃え |
| エラーメッセージ   | 「エラーが発生しました: {message}」形式 |
| メッセージスタイル | 下マージン3、赤色テキスト（text-red-700） |
| 再試行ボタン       | button（type="button"）、ラベル「再試行」 |

#### アクセシビリティ

| 属性          | 値      | 目的                               |
| ------------- | ------- | ---------------------------------- |
| role          | alert   | スクリーンリーダーに緊急通知として認識させる |
| button type   | button  | フォーム送信を防止                 |

## パフォーマンス

### 目標値

| 指標             | 目標   |
| ---------------- | ------ |
| 初期レンダリング | <100ms |
| リスト表示       | <200ms |
| 追加読み込み     | <100ms |
| フィルタ変更     | <150ms |

### 最適化戦略

| 戦略             | 適用場面                  |
| ---------------- | ------------------------- |
| ページネーション | 20件ずつ段階的読み込み    |
| メモ化           | useMemo/useCallbackで再計算防止 |
| 仮想スクロール   | 将来対応（大量データ時）  |
