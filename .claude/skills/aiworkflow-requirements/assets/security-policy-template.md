## 3. 入力検証

### 3.1 検証ルール

| フィールド | 検証項目         | ルール                |
| ---------- | ---------------- | --------------------- |
| id         | 形式             | UUID v4               |
| name       | 長さ、文字種     | 1-100文字、英数字記号 |
| email      | 形式             | RFC 5322準拠          |
| path       | パストラバーサル | 許可パス内のみ        |

### 3.2 サニタイズ

| 入力種別     | サニタイズ方法     |
| ------------ | ------------------ |
| HTML出力     | DOMPurify          |
| SQL          | パラメータバインド |
| ファイルパス | パス正規化+検証    |
| コマンド     | シェルエスケープ   |

### 3.3 Zodスキーマ例

```typescript
const InputSchema = z.object({
  name: z
    .string()
    .min(1)
    .max(100)
    .regex(/^[\w\s-]+$/),
  email: z.string().email(),
  path: z.string().refine(isValidPath),
});
```

### 4.1 IPC セキュリティ

| 設定             | 値    | 理由                 |
| ---------------- | ----- | -------------------- |

### 4.2 IPCチャンネル検証

```typescript
const ALLOWED_CHANNELS = [
  "resource:list",
  "resource:get",
  // ホワイトリスト
];

// Preloadでの検証
if (!ALLOWED_CHANNELS.includes(channel)) {
  throw new Error("Invalid IPC channel");
}
```

## 5. エラー処理

### 5.1 エラー情報の制御

| 環境 | エラー詳細 | スタックトレース |
| ---- | ---------- | ---------------- |
| 開発 | 表示       | 表示             |
| 本番 | 非表示     | 非表示           |

### 5.2 ログ記録

| イベント         | ログレベル | 記録項目                  |
| ---------------- | ---------- | ------------------------- |
| 認証成功         | info       | userId, timestamp         |
| 認証失敗         | warn       | ip, timestamp, reason     |
| 不正アクセス試行 | error      | ip, path, payload（一部） |

**禁止**: パスワード、トークン、APIキーのログ出力
